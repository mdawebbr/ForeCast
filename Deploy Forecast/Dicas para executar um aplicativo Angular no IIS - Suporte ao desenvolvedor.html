    Dicas para executar um aplicativo Angular no IIS

    Desenvolvedor

    14 de junho de 20173

    Laurie Atkinson , Consultora Sênior Premier Developer, Alguns ajustes são necessários para pegar um aplicativo Angular e movê-lo para o IIS. Aqui está uma lista de alterações necessárias para que tudo funcione sem problemas.

    Usar o Angular-CLI para gerar um SPA Angular é uma excelente maneira de desenvolver um aplicativo bem organizado. Essa ferramenta também inclui uma etapa de compilação de produção que deve permitir que você copie seu aplicativo para o servidor Web de sua escolha. Começaremos aqui e discutiremos as modificações necessárias para executar com êxito seu aplicativo no IIS.

    Use o Angular-CLI para criar um pacote de produção

    Primeiro decida se o aplicativo será localizado na raiz do IIS (por exemplo, http://server-name ) ou em um caminho abaixo da raiz (por exemplo , http://server-name/my-app ).

    Se estiver implantando na raiz, execute o comando Angular CLI build :

    Se estiver implantando em um caminho abaixo da raiz, execute o comando Angular CLI build com a opção base-href definida como o nome do diretório do aplicativo.

    A opção base-href modifica o elemento base no index.html , para que fique assim. Isso é necessário para que o roteamento funcione corretamente.

    Criar um aplicativo Web IIS

    Adicione o site, aplicativo ou diretório virtual ao IIS e defina o caminho físico para o local da pasta dist criada pela CLI Angular.

    Qualquer uma dessas três opções mostradas abaixo funcionará.

    Instale o módulo de reescrita de URL no IIS

    Esta próxima etapa é necessária para oferecer suporte a links diretos, que é a capacidade de o usuário navegar diretamente para uma página digitando a rota na barra de endereço em vez de usar o roteamento Angular. O link direto causa um problema para o IIS porque a URL que o usuário tenta acessar não é conhecida pelo servidor e, portanto, o usuário recebe uma resposta 404. A solução é que o servidor sempre retorne a raiz da aplicação, mesmo que o usuário solicite um caminho dentro da aplicação.

    Obtenha o instalador do módulo de reescrita de URL aqui: https://www.iis.net/downloads/microsoft/url-rewrite

    Após a instalação, você deverá ver um novo ícone no Gerenciador do IIS.

    Adicione o arquivo web.config com uma regra de reescrita de URL

    Todas as solicitações para este aplicativo da Web que não sejam para arquivos ou pastas devem ser direcionadas para a raiz do aplicativo. Para um aplicativo ou diretório virtual no site padrão, a URL deve ser definida como o alias (por exemplo, “/MyApp/”). Para um site na raiz do servidor, a URL deve ser definida como “/”.

    <configuration>

    <system.webServer>

      <rewrite>

        <rules>

          <rule name="Angular Routes" stopProcessing="true">

            <match url=".*" />

            <conditions logicalGrouping="MatchAll">

              <add input="{REQUEST_FILENAME}" matchType="IsFile" negate="true" />

              <add input="{REQUEST_FILENAME}" matchType="IsDirectory" negate="true" />

            </conditions>

            <action type="Rewrite" url="/MyApp/" />

            <!--<action type="Rewrite" url="/" />-->

          </rule>

        </rules>

      </rewrite>

    </system.webServer>

    </configuration>

    Solução de problemas

    Problema: Navegar diretamente para uma rota resulta em um erro 404.

    Solution: Make sure the web.config is correct and the URL property in the web.config matches both the base href in the index.html and the web application or virtual directory alias if necessary.

    Problem: Navigating directly to a route results in a 500 or an error message related to the web.config.

    Solution: Install the URL Rewrite module.

    Problem: Navigating directly to a route displays a broken page and network/console errors show 404 results for js and css requests.

    Solution: Verify that the base href in the index.html exactly matches the URL in the rewrite rule and that the slashes surrounding the name are not missing.
